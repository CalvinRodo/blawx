The Encoding Process

There are four steps in the encoding process:

  • Encode the vocabulary of your law
  • Use that vocabulary to encode facts, and rules
  • Test your code
  • Use your code

This tutorial will take you through all of those steps.

Step 1: Encoding Vocabulary

Blawx models the vocabulary of your law using categories and attributes. You create categories (like “dog”) with attributes (like “size” and “colour”), and then you can use those to describe objects (like “clifford”) with values (like “big” and “red”).

Open the navigation tree, and select section 1. It reads “A person over the age of 18 may make a will.”

We will model this as a single category for people, and two attributes for age and whether the user is entitled to make a will.

Step 1: The Person Category

To start, open the “Primary” drawer and drag a fact block onto the workspace.

Then open the “Categories” drawer and drag a new category block onto the workspace.

Now change the text in the new category block to read “person”. Note that it must start with a lowercase letter.

Click the “Save” button at the top of the screen before you continue.

Step 1: The Age Attribute

Now we need to describe for Blawx that the category of Person has an attribute called “age”.

Start by dragging a category attribute block from the Categories drawer, and adding it below the new category block. To tell this block what category we are adding attributes to, open the Known Categories drawer and connect the “person” category selector block.

To create the age attribute, open the “categories” drawer, drag out a new attribute block and put it inside the category attribute block. Then change the text for the name of the attribute from “attribute name” to “age”. Note that it must start with a lowercase letter.

Next, to indicate that the values in the age attribute should be numbers, open the “Data Types” drawer of the toolbox, and drag out and connect the “Number” datatype selector block.

Step 1: The Eligible Attribute

Now we need to describe for Blawx that the category of Person has an attribute called “eligible”.

To create the “eligible” attribute, open the “categories” drawer, and drag a new attribute block and put it inside the existing category attribute block below the “age” new attribute block. Now change the text for the name of the attribute from “attribute name” to “eligible”. Note that it must start with a lowercase letter.

Next, to indicate that the values in the age attribute should be either true or false, open the “Data Types” drawer of the toolbox, and drag out and connect the “True / False” datatype selector block.

Click the “Save” button at the top of the screen before you continue.

Step 2: Encoding Rules

Now that we have encoded the vocabulary present in section 1, we can encode the rule in section 1. Section 1 reads “A person over the age of 18 may make a will.” Because we have told Blawx about people, ages, and eligibility, we can create a rule that expresses this idea.

Start by opening the Primary drawer of the toolbox and dragging out an rule block pre-filled with an according to block, like this:

GIF

The rule starts with the conditions, and ends with the conclusions. The conditions of our rule are that there is a person over the age of 18. To express that idea in Blawx, we break it into three parts that are possible using the vocabulary provided.

  • Someone is a person
  • That someone has an age
  • That age is at least 18

The conclusion can be expressed like this:

  • That someone is eligible to make a will

Step 2: Rule Conditions

To encode the idea “someone is a person” as a condition, we will use an object category block from the Objects drawer, and add it to the conditions of our rule. We will then take the category selector for the “person” category from the Known Categories drawer, and a named variable from the Variables” drawer, and use them to fill in the inputs in the object category block. We will then change the name of the named Variable to “Someone”. It will look like this:

GIF

Note that the name of the variable must start with a capital letter.

To encode the idea “that someone has an age”, we will use the age attribute block we created in the Known Attributes drawer, with two named variable blocks from the Variables drawer, named “Someone” and “Age”, respectively. That looks like this:

GIF

To encode the idea “that age is at least 18”, we will use the numerical constraint block from the Data Statements / Number drawer of the toolbox, a named Variable from the variables drawer named “Age”, and a number value from the Data Values drawer set to 18. We will choose “greater than or the same value as” as the comparison type. That looks like this:

GIF
