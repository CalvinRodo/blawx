Step 3: Testing

Now that you have encoded the vocabulary, and used that vocabulary to encode rules, you can use the same vocabulary to set out a fact scenario, and ask a question about that fact scenario.

This can be used to see if the rules are behaving in the way expected.

In this example, we will create a person, give them an age, ask if they can make a will, and see what happens when we change their age.

Click the arrow on the right when you are ready to get started.

Step 3: Creating Objects and Attributes

To describe our fact scenario, that there is a person named Jane who is 12 years of age, we will drag a Fact block from the Primary drawer. Then we will add a new object block from the Objects drawer, and edit the field to give the new object the name "jane".

Note that the names of objects must start with lowercase letters.

To say that Jane is 12 years of age, we will drag the age attribute block from the Known Attributes drawer, and add it to the fact block. In its two inputs, we will add the "jane" object selector block from the known objects drawer, and a number value from the data Values drawer. The number field will then be set to 12.

Click "save" at the top of the screen, and then click on the right to continue.

Step 3: Asking Questions

Now that we have our fact scenario, we can ask whether jane is eligible to write a will. To do that, we grab a question block with an attacked holds block from the primary drawer. Then we drag the eligible attribute block from the known attributes drawer and add it to the holds block. For the two inputs we choose the true block from the data values drawer, and the jane object selector block from the known objects block.

Having posed the question, we can click "Save" at the top of the screen, and then click "Run" to have blawx answer the question. The answer will be "no models" as there is no way to prove the statement in the question is true. Click on the "Output" tab above to see that the answer is "No models", and then return to the tutorial tab.

Step 3: Getting Answers

Now let's see what happens when we change Jane's age to 20. Click on the number value block for Jane's age in the facts, and change it to 20. Click "Save", click "Run", and click on the "Output" tab above to see the response. An answer is provided with a natural language explanation, and links to the text of the rule that generated that result.

Step 4: Building An App

Now that we have written and tested our code, we can build an app.

Start by deleting the fact block from your test. This can be done by clicking on the fact block and clicking the "delete" key on your keyboard, or by right-clicking on the fact block and choosing "Delete 6 Blocks."

Now, delete the jane object selector block in the question, and replace it with a named variable. Set the name of the variable to "Someone". Note that the name must start with an uppercase letter.

Click "Save", and click "Bot". This will start a blawxbot user interface. If you give the answers "yes", "jane", "yes", and "20" in that order, (note that they all must be lowercase), Blawxbot will answer that Jane is eligible to make a will, and explain why.
